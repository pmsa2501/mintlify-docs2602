## Feature Overview

Ship Dispatch lets authorized staff record when cargo leaves the port by ship. When cargo is loaded onto a vessel, the system creates a **Buque despachado** (Ship Dispatched) entry to track the outgoing inventory movement.

<Note>
  Only authorized users (typically Admin and Operator) can create Ship Dispatch entries. Client users can view the Buques table but cannot create new dispatches.
</Note>

## Frequently Asked Questions

<AccordionGroup>
  <Accordion title="Why can't I create a dispatch?">
    If you see an error when saving, your account may not have permission to create dispatches. Contact your supervisor or system administrator.
  </Accordion>

  <Accordion title="Can I edit a dispatch after creating it?">
    The details panel is currently read-only. If you need to correct an entry, contact your system administrator.
  </Accordion>

  <Accordion title="What if I made a mistake in the quantity?">
    Contact your system administrator to create an adjustment (Ajuste) entry to correct the inventory balance.
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="I see 'No tiene acceso a las operaciones de PRC'">
    Your role does not have permission to create dispatches. Contact your supervisor.
  </Accordion>

  <Accordion title="The form won't submit">
    Make sure all required fields are filled: **Cliente**, **Producto**, **Cantidad**, **Motonave**.
  </Accordion>

  <Accordion title="My new entry doesn't appear">
    Refresh the page. If it still doesn't appear, contact technical support.
  </Accordion>
</AccordionGroup>

#### Important Notes

- All four fields (Cliente, Producto, Cantidad, Motonave) are required.
- Quantity must be a positive number in kilograms.
- Use the official vessel name for Motonave.
- Dispatch entries cannot be deleted through the interface; corrections must be made via adjustment entries.
