## Feature Overview

Every truck registered in MAM360 is always in exactly one of **13 estados**. The Estado Machine enforces which transitions are allowed, auto-assigns operational values like Turno numbers and timestamps, and restricts actions by role. Port operators never have to track state manually — the system handles it.

<Note>
  **PRC (Punto de Recepción y Control)** is the external reception area before the port facility. **IP (Instalación Portuaria)** is the port facility itself. For terminals without a PRC, trucks can skip estados 2–5 and enter directly at estado 4 (CITA agendada).
</Note>

## Special Notes

<AccordionGroup>
  <Accordion title="Estado 4 → 5 — ENVIAR Batch Dispatch">
    Estado 4 trucks are dispatched in bulk via the **ENVIAR** button in the PRC view — not through the estado dropdown.

    <Steps>
      <Step title="Open the Estado 4 pile">
        In the PRC view, open the pile with trucks in **Estado 4**.
      </Step>
      <Step title="Select trucks">
        Select the trucks to dispatch.
      </Step>
      <Step title="Dispatch">
        Click **ENVIAR**.
      </Step>
      <Step title="Review confirmation">
        A confirmation modal appears with the full truck list.
      </Step>
      <Step title="Confirm">
        Click **Continuar / Confirmar** — all selected trucks move to **Estado 5** and leave the PRC queue.
      </Step>
    </Steps>

    <Warning>
      ENVIAR is irreversible from the operator view. If dispatched in error, an Admin must correct the estado from the Camiones table.
    </Warning>
  </Accordion>

  <Accordion title="Estado X1 — Novedad Abierta (Problem Reporting)">
    When a truck cannot proceed normally, flag it as X1 from the PRC pile detail.

    <Steps>
      <Step title="Set estado to X1">
        Change the truck's estado to **X1 — Novedad abierta**.
      </Step>
      <Step title="Describe and set return">
        A modal opens with two required fields:
        - **Descripción del problema** — what went wrong.
        - **Estado de retorno** — where the truck resumes when resolved.
      </Step>
      <Step title="Submit">
        Submit. The truck is flagged and the return point is stored automatically.
      </Step>
      <Step title="Resolve or return">
        To resolve: change the estado to 2, 3, or 4 (as set in step 2).
      </Step>
      <Step title="Escalate if needed">
        If unresolvable, escalate to **X2 — Novedad cerrada** — this is terminal.
      </Step>
    </Steps>

    <Note>
      Trucks in X1 or X2 cannot be cancelled. Cancellation is only available from estados 2, 3, and 4.
    </Note>
  </Accordion>
</AccordionGroup>

## Architecture Notes

| Behavior | Reason |
|---|---|
| Default role = Cliente | Protects port operations by default. New users can only interact with estados 0–1 until an Admin explicitly grants higher access. |
| Time differentials auto-computed | Database triggers calculate elapsed time between every estado pair for KPI reporting. No manual entry required. |
